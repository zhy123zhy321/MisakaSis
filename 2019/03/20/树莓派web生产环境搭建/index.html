树莓派web生产环境搭建

2019-03-20 · 199 字 · 1 分钟
服务器
树莓派 · lnmp

网络配置

tf卡根目录中保存一个叫ssh的空文件，为了打开ssh功能。

再保存一个wpa_supplicant.conf

内容为：

country=CN 
ctrl_interface=DIR=/var/run/wpa_supplicant 
GROUP=netdev update_config=1 
network={ 
ssid="72change" 
psk="XXXXXXXXXX" 
key_mgmt=WPA-PSK 
priority=1 
} 
network={ 
ssid="TP-XXXX" 
psk="XXXXXXXX" 
key_mgmt=WPA-PSK 
priority=2 
}

priority是权重 谁高先连接谁。

基本环境配置

默认ssh 账户：pi密码：respberry

sudo raspi-config

进入树莓派bios界面

第一个是改pi的密码

进去选本地化的一项

里面有改时区，语言和键盘 。

开启root权限

sudo passwd root

输入两遍密码

sudo nano /etc/ssh/sshd_config

Ctrl + W 快捷键 搜索 PermitRootLogin

修改 PermitRootLogin without-password 为 PermitRootLogin yes

Ctrl + O 快捷键 保存

Ctrl + X 快捷键 退出 Nano 编辑器

sudo nano /etc/motd

这个里面是ssh 欢迎文字，可以全删改成自己喜欢的 这是一个文字字符画生成网站

更新树莓派系统

sudo apt update
sudo apt upgrade
sudo apt dist-upgrade

rdate 时间校对

因为树莓派没有电池，拔了电源时间就停了，所以要每次开机调整时间。

rdate的使用比较简单，先安装上rdate。

sudo apt-get install rdate
sudo rdate -s time.nist.gov

-s 是同步到本地，没有-s只是查看时间

添加开机启动

sudo nano /etc/rc.local

在exit0之前添加

rdate -s time.nist.gov

重启树莓派

sudo reboot

Lnmp 编译安装

ssh用root用户进入系统

apt-get install python-setuptools
wget -O install.sh http://download.bt.cn/install/install-ubuntu.sh && bash install.sh

安装宝塔

Bt-Panel: http://192.168.166.222:8888
username: 7bkdoe3g
password: 11c420ce

得到地址和用户名密码

登陆上去，开始安装

安装lnmp 一套除了mysql mysql 装不上，强行装还有很多问题很多坑。

解决方案1。编译安装mysql5.1。

只有这个版本树莓派装上没有错误，但是只能用phpMyAdmin操控，宝塔自带的数据库页面不可以使用。

解决方案2。在官方源里面装mysql-MariaDB。
apt-get install mysql-server
mysql -uroot
show databases;
use mysql;
select * from user where user = 'root'\G
看到密码是空的，改改
UPDATE user SET password=PASSWORD('你的密码') WHERE user='root';
select * from user where user = 'root'\G
改完再看看 Password: 加密了OK。
坑来了，真是大坑！

树莓派需要将用户表中plugin字段由auth_plugin设置成mysql_native_password，否则登陆不需要账户密码。

UPDATE user SET plugin='mysql_native_password' WHERE user='root';

FLUSH PRIVILEGES;

exit;

只有这个版本树莓派装上没有错误，但是只能用phpMyAdmin操控，同样宝塔自带的数据库页面不可以使用。

phpmyadmin如果出现远程连接问题：在/etc/mysql/mariadb.conf.d/50-server.conf中，将bind-address=127.0.0.1改成bind-address=0.0.0.0，重启系统即可。

phpmyadmin如果还连接不上就把phpmyadmin目录里config.inc.php，找到Servers host这里改成127.0.0.1

$cfg['Servers'][$i]['host'] = '127.0.0.1';
解决方案3。在官方源里面装轻量级的sqlite 代替mysql。
apt-get autoremove mysql-server
apt-get install php5-fpm php5-sqlite sqlite sqlite3

不是很推荐，比如上wordpress，还得装sqlite的插件才能连上sqlite数据库，麻烦啊~
